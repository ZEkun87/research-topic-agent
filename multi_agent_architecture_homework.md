# 多 Agent 协作架构说明（作业配套文字）

本科研选题 Agent 系统采用基于 LangGraph 的多 Agent 流水线协作架构，将复杂的科研选题任务拆分为多个专业化智能体，通过统一状态流转实现分工协作、分步决策，最终完成从用户需求到科研选题推荐的全流程自动化。

## 1. 多 Agent 分工与职责

系统共设计 5 个独立 Agent，每个 Agent 只负责单一核心任务，职责清晰、耦合度低，符合工程化设计原则：

### 需求解析 Agent

负责理解用户输入的研究兴趣与初步想法，通过大模型提取精准检索关键词，为后续文献检索提供准确入口。

### 文献检索 Agent

根据关键词自动调用学术 API（Semantic Scholar、arXiv），完成文献获取、去重、过滤与排序，保证数据源权威、时效性强。

### 领域分析 Agent

对检索到的文献进行归纳分析，提炼领域研究方向、近三年热点以及尚未解决的研究空白，为选题提供依据。

### 选题生成 Agent

结合领域现状与用户需求，自动生成 3 条结构完整的科研选题路径，每条包含选题名称、创新点、可行性分析及支撑文献。

### 收敛决策 Agent

对多条候选选题进行对比评估，综合创新性、可行性、学术价值给出最终推荐，并整理完整证据链，确保推荐结果可追溯、可验证。

## 2. 多 Agent 协作机制

各 Agent 之间不直接相互调用，而是通过共享的 `ResearchState` 状态对象进行数据传递。  
由 `StateGraph` 统一编排执行顺序，形成“需求解析 → 文献检索 → 领域分析 → 选题生成 → 收敛决策”的标准工作流。  
流程线性可控、逻辑清晰，便于调试、扩展与维护，是典型的工程化多 Agent 实现方式。

## 3. 实现效果总结

通过多 Agent 协作，系统最终实现：

- 用户输入一句话研究方向，全自动输出科研选题报告；
- 每条选题均有文献证据链支撑，可信度高；
- 具备明确的系统边界与能力范围，满足范围收敛要求；
- 架构可扩展，可后续加入人工验证、反馈迭代、多轮交互等模块。

该架构既体现了大模型应用的智能化能力，也符合 AI 系统工程化、模块化、可落地的设计思想。
